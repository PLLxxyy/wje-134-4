## 项目主要需求

### 项目概述

【纯后端服务】建筑工程项目成本管控 API 服务。面向建筑公司的财务和项目管理层，提供项目预算编制、成本归集、变更管理和成本分析的 RESTful API。**本项目用于验证模型跨文件协同改动能力，禁止将多个职责合并进单一文件。**

### 核心实体（≥ 3 个，每个贯穿后端全链路）

1. **项目预算（ProjectBudget）** — 字段：所属项目ID、预算名称、总预算金额、已用金额、预留金额、币种（CNY/USD）、审批状态（Draft/Submitted/Approved/Rejected）、审批人ID、审批日期、备注
   - 关联：1个预算 → 多个成本项
   - 业务动作：编制预算、提交审批、审批通过/驳回
   - **贯穿后端**

2. **成本项（CostItem）** — 字段：所属预算ID、成本类别（Material/Labor/Equipment/Subcontract/Overhead/Other）、成本名称、预算金额、实际金额、差异金额（自动计算）、发生日期、凭证号、关联材料领用ID（可选）、关联工时记录ID（可选）
   - 业务动作：录入成本、核对差异、标记异常
   - **贯穿后端**

3. **变更单（ChangeOrder）** — 字段：所属项目ID、变更类型（ScopeChange/DesignChange/PriceAdjustment/UnforeseenCondition）、变更描述、原预算金额、变更金额、变更后金额（自动计算）、申请理由、审批状态（Draft/Submitted/Approved/Rejected/Cancelled）、申请人ID、审批人ID、申请日期
   - 业务动作：发起变更、提交审批、审批/驳回、作废
   - **贯穿后端**

4. **成本分析报告（CostReport）** — 字段：所属项目ID、报告期间（月/季）、报告类型（Monthly/Quarterly/Annual/Custom）、人工成本汇总、材料成本汇总、设备成本汇总、其他成本汇总、总成本、盈亏分析、生成时间
   - **贯穿后端**

### 横切关注点（≥ 2 个）

1. **RBAC 权限控制**
   - 角色：Admin / FinanceManager / ProjectManager / Accountant / Viewer
   - 触达文件层：DB role表 + RBAC中间件 + 各controller权限检查 + 类型定义

2. **审计日志**
   - 记录预算审批、成本录入、变更审批等关键操作
   - 触达文件层：DB audit_log表 + 日志中间件 + 各service埋点 + 类型定义

3. **接口限流**
   - 触达文件层：限流中间件 + 配置文件 + Redis计数器

### 共享枚举/常量（≥ 2 组）

1. **预算审批状态 BudgetStatus**：Draft / Submitted / Approved / Rejected
   - 定义：`backend/src/types/enums.ts`

2. **成本类别 CostCategory**：Material / Labor / Equipment / Subcontract / Overhead / Other
   - 定义：`backend/src/types/enums.ts`

3. **变更类型 ChangeType**：ScopeChange / DesignChange / PriceAdjustment / UnforeseenCondition

4. **变更审批状态 ChangeOrderStatus**：Draft / Submitted / Approved / Rejected / Cancelled

### 后端中间件清单（≥ 2 个）

1. **`authMiddleware`** — JWT 鉴权
2. **`rbacMiddleware`** — 角色权限校验（只有FinanceManager能审批预算，只有Accountant能录入成本）
3. **`auditLogMiddleware`** — 审计日志
4. **`errorHandlerMiddleware`** — 统一异常
5. **`rateLimitMiddleware`** — 接口限流（基于Redis，每IP每分钟60次）
6. **`requestLoggerMiddleware`** — 请求日志

### 技术要求

- **英文短名（用于 Docker Compose）**：`cost-control-api`
- **后端端口**：19203 / 3000
- **后端**：NestJS + TypeScript
- **数据库**：PostgreSQL 15
- **ORM**：TypeORM
- **缓存**：Redis 7（限流 + 报表缓存）
- **认证**：JWT
- **API文档**：Swagger（自动生成）

### 部署要求

后端 + 数据库 + Redis，Docker Compose 编排。

docker-compose.yml 要求：
- 顶层 `name: cost-control-api`
- 不写 `version:` 字段
- 容器名带 `${COMPOSE_PROJECT_NAME:-cost-control-api}` 前缀
- PostgreSQL + Redis 均使用命名卷
- PostgreSQL `healthcheck` + 后端 `depends_on.service_healthy`
- 后端端口 `${BACKEND_PORT:-3000}:3000`
- `.env` / `.env.example` 必含：`COMPOSE_PROJECT_NAME`, `DB_NAME`, `DB_USER`, `DB_PASSWORD`, `DB_ROOT_PASSWORD`, `JWT_SECRET`, `REDIS_URL`, `BACKEND_PORT`

### Git 要求

`git init` + `.gitignore`（node_modules/、dist/、.env、.DS_Store、*.log、.idea/、.vscode/）。
